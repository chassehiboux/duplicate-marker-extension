# План адаптации Firefox

1. Создать отдельную ветку `firefox-adaptation`.
2. Установить и проверить `web-ext`.
3. Добавить генератор Firefox-сборки.
4. Подключить Firefox-режим к `deploy.py`.
5. Сделать Firefox-совместимый manifest.
6. Добавить совместимость панели:
   - Chrome: `chrome.sidePanel`;
   - Firefox: `browser.sidebarAction` / `chrome.sidebarAction`.
7. Отключить в Firefox-сборке Google ITIL-модуль, завязанный на неподдерживаемый Firefox API `chrome.debugger`, с понятным сообщением в UI.
8. Проверить, что StageTimer остается синхронизированным между основной и обособленной сборками.
9. Собрать `dist/firefox/`.
10. Запустить `web-ext lint --source-dir dist/firefox`.
11. Проверить проект на признаки mojibake.
12. Создать отдельный коммит только с изменениями Firefox-адаптации.

## Принятые дефолты

- Не создавать отдельную копию исходников под Firefox.
- Не ломать текущий Chrome-деплой.
- Firefox-сборка считается артефактом и пересоздается скриптом.
- Для сетевого деплоя использовать отдельную папку, чтобы пользователи Chrome и Firefox не ставили разные manifest из одной директории.
