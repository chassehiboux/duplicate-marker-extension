# Quickstart проверки Supabase-синхронизации

## Первый браузер

1. Установить расширение с локальными данными.
2. Войти в Supabase.
3. Проверить, что появилось устройство, server revision больше `0`, очередь изменений пустая.
4. Проверить, что локальные настройки и счетчики совпадают с canonical state.

## Второй браузер

1. Установить расширение во второй профиль браузера.
2. До входа создать отличающиеся локальные настройки.
3. Войти в тот же Supabase-аккаунт.
4. Проверить, что локальные отличающиеся данные не отправились поверх облака.
5. Проверить, что локальный кэш заменен состоянием из Supabase.
6. Проверить наличие pre-login backup.

## Realtime

1. Открыть два браузера с одним аккаунтом.
2. В первом изменить настройку или тег действия.
3. Проверить, что во втором браузере изменение появилось без ручной синхронизации.
4. Проверить, что статус показывает успешный pull и актуальные revision.

## NetworkError

1. Отключить сеть или заблокировать Supabase REST.
2. Изменить настройку.
3. Проверить, что изменение остается в outbox.
4. Проверить, что UI показывает ошибку push и размер очереди.
5. Вернуть сеть.
6. Проверить, что outbox отправился автоматически.

## Счетчики

1. В двух браузерах увеличить один и тот же счетчик за день.
2. Проверить, что оба изменения учтены.
3. Проверить, что итоговое значение не потеряло один из increment.

## Импорт JSON

1. Без входа выполнить импорт и убедиться, что он остается локальным.
2. После входа выполнить импорт и подтвердить обновление облачных данных.
3. Проверить, что импорт прошел через mutation, а затем применился canonical state.

## Старый snapshot

1. Поискать активные вызовы `upsertRemoteState`.
2. Проверить, что изменения sync-ключей, ручная синхронизация, импорт и realtime не пишут полный `extension_user_state.state`.
