# Классификация заявок через расширение

## Спецификация

Цель: заменить медленную HtmlService-модалку `problemPicker.html` локальной модалкой расширения на странице Google Sheets и вынести действия модуля в меню расширения по `F2`.

Таблица: `1A7bsMpcMegLvTIpdRBa06TD5RxGi64Cw3goYydbujNo`.

Код расширения вынесен в отдельный блок:
- `GoogleSheets/problem_picker_config.js`
- `GoogleSheets/problem_picker_content.js`
- `GoogleSheets/problem_picker_background.js`

Модуль подключается отдельной записью `content_scripts` только для URL:

`https://docs.google.com/spreadsheets/d/1A7bsMpcMegLvTIpdRBa06TD5RxGi64Cw3goYydbujNo/*`

Листы для классификации:
- `Заявки`
- `Заявки (наши)`

Колонки определяются по заголовкам:
- `Текст заявки`
- `Проблемы`

Очередь формируется в расширении по CSV-экспорту активного листа: берутся строки, где `Текст заявки` заполнен, а `Проблемы` пустая. На реальной таблице проверено: экспорт листа `Заявки` возвращает 2545 строк, `Проблемы` находится в колонке B, `Текст заявки` в колонке T.

## Ограничение записи

Запись через UI Google Sheets не используется. На реальной таблице включён фильтр: при переходе к `B2021` интерфейс выбрал `B2029`, поэтому автоматическая вставка через DOM может записать значение в неверную строку.

Для сохранения используется лёгкий Apps Script bridge:
- расширение показывает модалку, хранит очередь и быстро переключает заявки;
- bridge принимает `sheetName`, `row`, `value`;
- bridge сохраняет категорию и принимает уже очищенный расширением текст заявки, чтобы Apps Script не зависел от серверной функции очистки текста.

## План

1. Добавить отдельное меню расширения по `F2` на странице указанной таблицы.
2. Загружать CSV активного листа через `fetch(..., { credentials: 'omit' })`.
3. Показывать локальную модалку классификации с теми же вариантами проблем.
4. Сохранять выбор через bridge URL, заданный кнопкой `URL bridge` в меню `F2`.
5. Скрывать элементы модалки при создании скриншота.
6. Форматировать `Текст заявки` на стороне расширения до отправки в bridge.

## Меню F2

На странице Google Sheets модуль показывает отдельное меню по `F2`.

Во вкладке `Функции`:
- `Классификация` открывает локальную модалку выбора проблемы.
- `URL bridge` задаёт URL Apps Script Web app для записи в таблицу.
- `ITIL: заполнить текст заявок` содержит кнопки `Старт` и `Стоп`, счётчики очереди и строку текущей обработки.

## Заполнение текста из ITIL

Очередь ITIL формируется из CSV активного листа `Заявки` или `Заявки (наши)` по заголовкам:
- `Номер ITIL`
- `Номер СУПП (последний)`
- `Текст заявки`

В очередь попадают строки, где `Номер СУПП (последний)` равен `–`, `Текст заявки` пустой, а `Номер ITIL` заполнен.

Процесс:
1. Проверить наличие вкладки `http://tmn-vpitil1c01/ITIL/ru_RU/`; если её нет, открыть.
2. Дождаться личного кабинета и открыть `Service Desk`, если вкладка ещё не открыта.
3. Для каждого номера ITIL открыть окно `Найти` через `Alt+F`, выбрать поле `Номер`, заполнить `Что искать` номером ITIL и нажать `Найти`.
4. Дождаться единственной найденной строки, открыть заявку двойным кликом.
5. Считать текст из iframe полного текста/переписки и значение поля `Решение обращения`.
6. Очистить текст заявки той же логикой, которая раньше была в `Code.gs`: взять первый ITIL-комментарий между строками автора, а если найдена только одна авторская строка, взять текст после неё до конца; убрать строки с именами файлов, вырезать блок по маркерам `Детальное описание:` / `Причина возникновения инцидента:`, удалить `ЗНО ...`, нормализовать пробелы и пустые строки.
7. Записать в таблицу:
   - `Текст заявки` = отформатированный текст из ITIL;
   - `Информация из СУПП/ITIL` = `ИТИЛ\n\n` + текст решения.
8. Закрыть вкладку заявки и перейти к следующему номеру.

Кнопка `Стоп` прекращает дальнейшую обработку и сбрасывает runtime-состояние очереди. Долговечные ключи `chrome.storage.local` для ITIL-очереди не используются.

Быстрая строка поиска Service Desk и системный clipboard больше не используются: фильтрация идёт через штатное окно `Найти`, чтобы одинаково работать в Chrome и Firefox.

## Быстрый старт

1. Добавить файл `ProblemPickerBridge.gs` в Apps Script проект таблицы.
2. Задеплоить Apps Script как Web app.
3. Открыть таблицу, нажать `F2` и выбрать `URL bridge`.
4. Вставить URL Web app вида `https://script.google.com/macros/s/.../exec`.
5. Нажать `F2` и выбрать `Классификация` или `Старт` в блоке ITIL.
