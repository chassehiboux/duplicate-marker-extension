# StageJump: фиксированные правила

## 1) Размещение кнопки перехода
- Кнопка StageJump (`dup-stage-jump-btn`) ставится в ячейку `cbox` строки:
  - `td[aria-describedby="list_cb"]`
- Отдельные технические колонки под кнопку не создаются.
- В строке кнопка заменяет штатный `cbox`-контрол.

## 2) Автофильтр DebtID (one-shot)
- Применение фильтра DebtID выполняется ровно один раз на страницу.
- Перед применением обязательно ожидать StageTimer:
  - обязательно дождаться факта состояния `Загрузка` у StageTimer;
  - применять автофильтр только после завершения этой `Загрузка`;
  - состояние `Фильтрация` не является триггером запуска;
  - если `Загрузка` не была зафиксирована (скрипт подключился поздно), ждать ограниченное время и затем запускать one-shot без повторных каскадов.
- Для отправки фильтра использовать один путь:
  - `setGridParam(...filters...)` + `reloadGrid(...)`
- Не использовать комбинации `triggerToolbar`/`Enter`/дополнительные триггеры, чтобы не запускать каскад повторных фильтраций.

## 3) Критерий завершения
- После one-shot применения:
  - дождаться завершения запроса;
  - проверить, что DebtID-фильтр реально закрепился в `postData.filters`.
- Если не закрепился за таймаут — завершать попытку с ошибкой, без повторного каскадного применения.
